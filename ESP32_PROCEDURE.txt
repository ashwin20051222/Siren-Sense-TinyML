# Step-by-Step Procedure: Using Local AI with ESP32-CAM + ESP-NOW
## Siren Sense — Emergency Sound Detection + Ambulance Verification

---

## WHAT THIS DOES

Your laptop microphone listens for ambulance sirens using a trained AI model.
When a siren is detected, your laptop tells the ESP32-CAM (over WiFi) to take
a photo. The ESP32 sends that photo to Roboflow to check if it's really an
ambulance. If yes, the ESP32-CAM sends an ESP-NOW message to a second ESP32
(receiver), which forwards it via UART wire to the STM32 traffic controller.

```
Laptop Mic → AI Model → WiFi → ESP32-CAM → Camera → Roboflow → ESP-NOW → Receiver ESP32 → UART → STM32
```

---

## PREREQUISITES

Hardware:
  - Laptop with microphone (built-in or USB)
  - ESP32-CAM (AI-Thinker) with OV2640 camera
  - ESP32 DevKit (any standard ESP32 board) — the receiver
  - Sound sensor module (analog, optional fallback)
  - 16x2 I2C LCD with PCF8574 backpack (optional status display, up to 3)
  - FTDI USB-UART adapter (for flashing ESP32-CAM)
  - Jumper wires (3 wires for ESP32 receiver → STM32 connection)

Software:
  - Python 3.10 / 3.11 / 3.12 on laptop
  - Arduino IDE (with ESP32 board package installed)
  - Roboflow account with trained ambulance detection model

Both laptop and ESP32-CAM must be on the SAME WiFi network.
Both ESP32 boards must be on the SAME WiFi network (for ESP-NOW channel).

---

## STEP 1: WIRE THE HARDWARE

Receiver ESP32 → STM32 (USART3 — only 3 wires!):
  ESP32 GPIO 17 (TX2) → STM32 PB11 (USART3 RX)
  ESP32 GPIO 16 (RX2) → STM32 PB10 (USART3 TX)   [optional]
  ESP32 GND            → STM32 GND                 [MUST connect!]

Sound Sensor → ESP32-CAM (optional fallback):
  Sensor VCC   → ESP32-CAM 3.3V
  Sensor GND   → ESP32-CAM GND
  Sensor OUT   → ESP32-CAM GPIO 33

LCD 16x2 I2C → ESP32-CAM (optional status display):
  LCD SDA      → ESP32-CAM GPIO 26     (shared with camera, uses Wire1)
  LCD SCL      → ESP32-CAM GPIO 27     (shared with camera, uses Wire1)
  LCD VCC      → ESP32-CAM 5V
  LCD GND      → ESP32-CAM GND
  (Default I2C address: 0x27. Try 0x3F if display is blank.)
  (Set LCD_ENABLED to 0 in the code if no LCD connected.)

LCD 16x2 I2C → Receiver ESP32 (optional):
  LCD SDA      → ESP32 GPIO 21
  LCD SCL      → ESP32 GPIO 22
  LCD VCC      → ESP32 5V
  LCD GND      → ESP32 GND

FTDI Programmer → ESP32-CAM (for flashing only):
  FTDI 3.3V    → ESP32-CAM 3.3V
  FTDI GND     → ESP32-CAM GND
  FTDI TX      → ESP32-CAM GPIO 3 (RX0)
  FTDI RX      → ESP32-CAM GPIO 1 (TX0)
  Jumper wire: ESP32-CAM IO0 → GND  (ONLY during flashing)

## STEP 3: INSTALL ARDUINO LIBRARY

Open Arduino IDE → Sketch → Include Library → Manage Libraries:

  1. Search "ArduinoJson" → install "ArduinoJson by Benoit Blanchon" (v6 or v7)
  2. ESP32 board package should already be installed

Note: No LoRa library needed! ESP-NOW is built into the ESP32.

---

## STEP 4: FLASH THE RECEIVER ESP32 (DO THIS FIRST!)

Why first? You need the receiver's MAC address before you can configure
the sender.

1. Open the file:
   esp32_firmware/stm32_esp32_receiver/stm32_esp32_receiver.ino

2. Edit Section 1 — only WiFi credentials needed:
   #define WIFI_SSID           "YourWiFiName"        ← your WiFi name
   #define WIFI_PASSWORD       "YourWiFiPassword"    ← your WiFi password

3. In Arduino IDE:
   - Board:            "ESP32 Dev Module" (or your specific DevKit board)
   - Upload Speed:      115200
   - Port:              your USB port (e.g. /dev/ttyUSB0)

4. Click Upload
5. Wait for "Done uploading"

6. Open Serial Monitor at 115200 baud
7. Press Reset on the ESP32
8. Look for this output:

   ┌─────────────────────────────────────────────┐
   │  THIS ESP32 MAC: AA:BB:CC:DD:EE:FF          │
   │  ↑↑ Copy this to sender's RECEIVER_MAC! ↑↑  │
   └─────────────────────────────────────────────┘

9. WRITE DOWN THIS MAC ADDRESS — you need it in Step 5.

---

## STEP 5: CONFIGURE AND FLASH THE ESP32-CAM (SENDER)

1. Open the file:
   esp32_firmware/esp32_cam_sender/esp32_cam_sender.ino

2. Edit Section 1 (User Configuration) near the top:

   #define WIFI_SSID           "YourWiFiName"        ← SAME WiFi as receiver!
   #define WIFI_PASSWORD       "YourWiFiPassword"    ← your WiFi password
   #define ROBOFLOW_API_URL    "https://detect.roboflow.com/YOUR_PROJECT/VERSION"
   #define ROBOFLOW_API_KEY    "YOUR_API_KEY"
   #define ROBOFLOW_TARGET_CLASS "ambulance"          ← class name from Roboflow
   #define MONITORED_LANE_ID   0                      ← lane (0=N, 1=E, 2=S, 3=W)
   #define LCD_ENABLED         1                      ← 0 if no LCD connected

   And most importantly — set the receiver's MAC address from Step 4:
   static uint8_t RECEIVER_MAC[] = {0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF};
   ← Replace with YOUR receiver's MAC!

   Example: If receiver MAC is 24:6F:28:AB:CD:EF, set:
   static uint8_t RECEIVER_MAC[] = {0x24, 0x6F, 0x28, 0xAB, 0xCD, 0xEF};

3. Save the file.

4. In Arduino IDE:
   - Board:            "AI Thinker ESP32-CAM"
   - Partition Scheme:  "Huge APP (3MB No OTA / 1MB SPIFFS)"
   - Upload Speed:      115200
   - Port:              your FTDI adapter (e.g. /dev/ttyUSB0)

5. Connect IO0 to GND on ESP32-CAM (flash mode)
6. Press Reset button on ESP32-CAM
7. Click Upload in Arduino IDE
8. Wait for "Done uploading"
9. DISCONNECT IO0 from GND
10. Press Reset button again

---

## STEP 6: FIND THE ESP32-CAM IP ADDRESS

1. In Arduino IDE → Tools → Serial Monitor
2. Set baud rate to 115200
3. Press Reset on ESP32-CAM
4. Look for this line in the output:

   [WiFi] Connected! IP: 192.168.1.50
   [HTTP] Trigger URL: http://192.168.1.50/trigger

5. WRITE DOWN THIS IP ADDRESS — you need it in Step 8.

   You should also see:
   [ESP-NOW] Initialized. Receiver: AA:BB:CC:DD:EE:FF

   If you see "[WiFi] FAILED to connect!":
   - Double-check WIFI_SSID and WIFI_PASSWORD in the code
   - Make sure it's a 2.4 GHz network (ESP32 doesn't support 5 GHz)

## STEP 7: SET UP THE LAPTOP

1. Open a terminal on your laptop.

2. Navigate to the project:
   cd "/home/silicondoc/Documents/Fun Project/AI ML For audio"

3. Create and activate the Python virtual environment:
   python3 -m venv venv
   source venv/bin/activate

4. Install dependencies:
   pip install -r requirements_pc.txt

5. Find your laptop's IP (just to verify you're on the same network):
   hostname -I
   → Example output: 192.168.1.105

   Your laptop IP and ESP32-CAM IP should start with the same numbers
   (e.g. both start with 192.168.1.xxx). If not, they're on
   different networks and this won't work.

## STEP 8: RUN THE LOCAL AI + ESP32 SYSTEM

Make sure:
  ✓ Receiver ESP32 is powered and connected to WiFi (Step 4)
  ✓ Receiver ESP32 is wired to STM32 (Step 1)
  ✓ ESP32-CAM is powered on and connected to WiFi (Step 6)
  ✓ Laptop venv is activated (Step 7)
  ✓ You have the ESP32-CAM IP address from Step 6

Run this command (replace the IP with YOUR ESP32-CAM's IP):

  python3 src/siren_to_esp32.py --mode wifi --esp32-ip 192.168.1.50 --threshold 0.80

You should see:
  ┌──────────────────────────────────────────────────────┐
  │  🚨  Siren Sense ML → ESP32-CAM Ambulance Detector  │
  │  🎯  Target label  : emergency                       │
  │  📊  Threshold     : 80%                              │
  │  📡  Trigger mode  : wifi                             │
  │  🌐  ESP32-CAM IP  : 192.168.1.50:80                 │
  │  🛑  Press Ctrl+C to stop                             │
  └──────────────────────────────────────────────────────┘

The system is now running! It continuously:
  - Listens through your laptop microphone
  - Runs the AI model every 1 second
  - Shows live probabilities on screen

## STEP 10: STOPPING THE SYSTEM

- On laptop: Press Ctrl+C to stop the Python script
- ESP32-CAM will keep running (sound sensor fallback still active)
- Receiver ESP32 keeps running (waiting for ESP-NOW messages)
- To restart: run the Step 8 command again

## COMMON PROBLEMS AND FIXES

Problem: "ESP32-CAM not reachable"
Fix:    - Is ESP32-CAM powered on?
        - Is it on the same WiFi as your laptop?
        - Check the IP in Serial Monitor
        - Try: curl http://192.168.1.50/health

Problem: "No audio input" or empty predictions
Fix:    - Run: python3 src/inference_pi.py --list-devices
        - Make sure a microphone is connected / not muted
        - Check system audio settings

Problem: Roboflow returns error on ESP32-CAM
Fix:    - Check ROBOFLOW_API_KEY and ROBOFLOW_API_URL in the .ino
        - Make sure ESP32-CAM has internet access (not just local WiFi)
        - Try the API URL in a browser first

Problem: ESP-NOW send fails
Fix:    - Both ESP32s must be on the SAME WiFi network
        - Double-check the receiver MAC address in sender code
        - Make sure WiFi is 2.4 GHz (not 5 GHz)

Problem: Receiver not getting messages
Fix:    - Verify MAC address (printed on receiver Serial Monitor)
        - Ensure both ESP32s are powered and WiFi connected
        - Check that both use the same WiFi SSID

Problem: STM32 not responding to alerts
Fix:    - Check UART wiring: ESP32 GPIO 17 → STM32 PB11
        - Check GND is connected between ESP32 and STM32
        - Open STM32 serial monitor to see if messages arrive

Problem: LCD is blank or not showing anything
Fix:    - Check SDA/SCL wiring
        - Try changing LCD_I2C_ADDR to 0x3F in the code
        - Set LCD_ENABLED to 0 if you don't have an LCD

Problem: Permission denied on /dev/ttyUSB0
Fix:    - Run: sudo usermod -aG dialout $USER
        - Log out and log back in

Problem: ESP32-CAM says "[WiFi] FAILED"
Fix:    - ESP32 only works with 2.4 GHz WiFi (not 5 GHz)
        - Check SSID and password — they are case sensitive
